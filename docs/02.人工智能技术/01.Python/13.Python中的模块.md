# Python 中的模块

在 Python 编程中，随着项目代码量的不断增加，将所有逻辑堆砌在一个文件中会变得难以维护。这时，“模块”就成为了我们组织代码、提高复用性的核心利器。

---

### 什么是模块？

简单来说，模块就是一个包含 Python 代码的 `.py` 文件。它就像一个装满工具的箱子，里面可以封装函数、类以及变量。 使用模块主要有三大好处：

1. **代码组织**：将相关的功能逻辑拆分到不同的文件中，使项目结构更清晰。
2. **代码复用**：写好一个模块后，可以在不同的项目中反复调用，避免重复造轮子。
3. **命名空间管理**：不同模块中可以有相同名称的函数或变量，互不冲突。

---

### 模块的导入方式

Python 提供了多种灵活的导入方式，以适应不同的使用场景。假设我们有一个名为 `math_utils.py` 的模块，里面定义了 `add(a, b)` 函数：

- **导入整个模块**：调用时需要加上模块名前缀，能有效避免命名冲突。

  ```python
  import math_utils
  print(math_utils.add(10, 5))
  ```

- **导入模块并指定别名**：当模块名过长时，可以使用 `as` 关键字简化调用。

  ```python
  import math_utils as mu
  print(mu.add(10, 5))
  ```

- **从模块中导入特定函数**：直接导入需要的功能，调用时无需加前缀。

  ```python
  from math_utils import add
  print(add(10, 5))
  ```

- **导入模块所有内容（不推荐）**：虽然方便，但极易引发命名冲突，导致代码难以排查。

  ```python
  from math_utils import *
  ```

---

### Python 模块的三大分类

在实际开发中，我们会接触到三种不同来源的模块：

1. **内置模块（标准库）**：Python 安装时自带的模块，无需额外安装，功能强大且性能高效。
   - `math`：提供数学运算函数（如 `math.sqrt()` 开平方）。
   - `os` / `sys`：用于与操作系统交互（如文件路径管理、获取 Python 环境变量）。
   - `random` / `time` / `datetime`：处理随机数与时间日期。
2. **自定义模块**：程序员自己编写的 `.py` 文件，用于封装项目特有的业务逻辑。
3. **第三方模块**：由全球开发者社区贡献的强大工具库，通常需要通过包管理工具 `pip` 进行安装。
   - 例如：用于数据分析的 `pandas`、用于网络请求的 `requests`、用于科学计算的 `numpy` 等。

### 常用内置模块

#### 时间模块

##### **1. datetime 模块：业务开发中的时间魔法师**

`datetime` 是日常业务开发（如订单有效期、用户活跃统计）中最常用的模块。它主打面向对象，提供了多种针对不同业务场景的时间表示方式。

**1.1 基础三剑客：date、time 与 datetime**

- **date（纯日期）**：只包含年、月、日。适用于生日、节假日等场景。
- **time（纯时间）**：只包含时、分、秒、微秒。适用于表示每天固定的时刻（如打卡时间）。
- **datetime（日期+时间）**：前两者的结合体，包含了完整的年月日时分秒信息，是业务系统中最默认采用的形式。

```python
from datetime import date, time, datetime

# 1. date：仅表示日期
birthday = date(1998, 7, 4)
print("生日日期：", birthday)  # 输出: 1998-07-04

# 2. time：仅表示时间
work_start = time(9, 0, 0)
print("上班时间：", work_start)  # 输出: 09:00:00

# 3. datetime：日期与时间的结合
meeting = datetime(2026, 5, 8, 14, 30)
print("会议时间：", meeting)  # 输出: 2026-05-08 14:30:00
```

**1.2 强大的时间加减（timedelta）**
通过 `timedelta`，可以轻松计算 “3 天 5 小时后” 是什么时间，或者计算两个日期的时间差。

```python
from datetime import datetime, timedelta

now = datetime.now()
# 计算 3 天后的时间
future = now + timedelta(days=3, hours=5)
print("3天5小时后：", future.strftime("%Y-%m-%d %H:%M:%S"))
# 输出: 3天5小时后： 2026-05-11 21:05:48

# 计算两个时间的差值
start = datetime(2026, 5, 1, 9, 0, 0)
end = datetime(2026, 5, 1, 18, 30, 0)
diff = end - start
print("工作时长（秒）：", diff.total_seconds()) 
# 输出: 工作时长（秒）： 34200.0
```

**1.3 国际化标准：ISO 8601 格式**
在跨系统交互、API 接口对接或日志记录时，通过 `isoformat()` 方法可以将时间对象转换为全球通用的字符串格式（如 `YYYY-MM-DDTHH:MM:SS`）。

```python
from datetime import datetime

now = datetime.now()
# 转换为 ISO 8601 标准格式字符串
iso_str = now.isoformat()
print("ISO格式时间：", iso_str)  
```

**1.4 带时区的时间（Aware Datetime）**
默认生成的 `datetime` 对象是“天真”（Naive）的，不包含时区信息。处理跨国业务时，建议在 Python 3.9+ 中结合 `zoneinfo` 模块生成带时区的“感知”（Aware）时间对象。

```python
from datetime import datetime
from zoneinfo import ZoneInfo

# 生成带上海时区信息的时间对象
shanghai_time = datetime.now(ZoneInfo("Asia/Shanghai"))
print("上海带时区时间：", shanghai_time)
# 输出示例: 2026-05-08 14:36:36.123456+08:00
```

##### **2. time 模块：底层时间戳、struct_time 与性能计时专家**

`time` 模块更偏向底层，主要围绕从 1970 年 1 月 1 日开始的秒数（Unix 时间戳）进行操作，适合处理系统时间、程序延时和高精度计时。

**2.1 核心概念：时间戳与 struct_time 的转换**
`struct_time` 是 `time` 模块中一种核心的时间表示形式，它像一个带有属性名的元组，把时间信息拆解成了 9 个具体的元素。

- `time.time()`：获取当前时间戳（浮点数）。
- `time.localtime()`：把时间戳转换成本地时间的 `struct_time` 元组。
- `time.gmtime()`：把时间戳转换成 UTC（世界标准时间）的 `struct_time` 元组。
- `time.mktime()`：把本地时间的 `struct_time` 元组转换回时间戳。

`struct_time` 包含以下 9 个核心属性：

- `tm_year`（年）、`tm_mon`（月，1-12）、`tm_mday`（日，1-31）
- `tm_hour`（时，0-23）、`tm_min`（分，0-59）、`tm_sec`（秒，0-61）
- `tm_wday`（星期几，0-6，**0代表星期一**）
- `tm_yday`（一年中的第几天）、`tm_isdst`（是否夏令时）

**2.2 实战演练：struct_time 的提取与转换**

```python
import time

# 获取当前的 struct_time（本地时间）
current_struct = time.localtime()
print("当前的 struct_time：", current_struct)
# 输出: 当前的 struct_time： time.struct_time(tm_year=2026, tm_mon=5, tm_mday=11, tm_hour=10, tm_min=57, tm_sec=32, tm_wday=0, tm_yday=131, tm_isdst=0)

# 通过属性名提取具体信息
print(f"当前是 {current_struct.tm_year} 年")
print(f"今天是星期 {current_struct.tm_wday + 1}") # 0代表周一，所以+1
# 输出:
# 当前是 2026 年
# 今天是星期 1

# 将 struct_time 转换为格式化字符串（strftime）
formatted_str = time.strftime("%Y-%m-%d %H:%M:%S", current_struct)
print("格式化时间：", formatted_str)
# 输出: 格式化时间： 2026-05-11 10:57:32

# 将格式化字符串解析回 struct_time（strptime）
parsed_struct = time.strptime("2026-05-08 14:30:00", "%Y-%m-%d %H:%M:%S")

# 将 struct_time 转换回时间戳
timestamp = time.mktime(parsed_struct)
```

**2.3 常用方法与代码示例**

- **获取绝对时间戳**：`time.time()` 常用于记录事件发生的绝对时间。
- **程序延时**：`time.sleep(secs)` 用于让当前线程暂停执行指定的秒数。
- **快速转换为可读字符串**：`time.ctime()` 或 `time.asctime()` 可以快速将时间戳或元组转为易读的字符串格式。
- **单调时钟**：`time.monotonic()` 返回一个绝对不会倒退的时钟值，即使手动修改系统时间也不会受影响。

```python
import time

# 1. 获取当前时间戳
print(f"当前时间戳: {time.time()}")

# 2. 程序延时 1 秒
print("准备暂停 1 秒...")
time.sleep(1)
print("暂停结束！")

# 3. 快速转为可读字符串
print(f"当前时间 (ctime): {time.ctime()}")

# 4. 单调时钟测量长时间任务
start = time.monotonic()
time.sleep(0.5) # 模拟耗时任务
end = time.monotonic()
print(f"任务实际耗时: {end - start:.2f} 秒")
```

**2.4 高精度性能计时**
测试代码执行耗时，**强烈推荐使用 `time.perf_counter()`**，而不是 `time.time()`。`perf_counter` 提供了系统所能提供的最高精度计时器，专门用于测量短时间间隔。

```python
import time

# 记录开始时间
start = time.perf_counter()
# 模拟一段耗时操作
for _ in range(1000000):
    pass
# 记录结束时间并计算差值
end = time.perf_counter()
print(f"代码执行耗时：{end - start:.6f} 秒")
```

##### **3. calendar 模块：日历算法与周期性任务神器**

当业务涉及到“周期性”或者“日历展示”时（比如生成打卡日历、判断某个月有多少个工作日），`calendar` 模块就派上用场了。

**3.1 获取某个月的天数和起始星期**
`calendar.monthrange(year, month)` 会返回一个元组：`(该月第一天是星期几, 该月总天数)`。注意，它默认 **0 代表星期一，6 代表星期日**。

```python
import calendar

# 查询 2026 年 5 月的信息
first_weekday, days_count = calendar.monthrange(2026, 5)
print(f"2026年5月共有 {days_count} 天")
print(f"5月1日是星期 {first_weekday + 1}") # +1 是为了符合我们日常说的星期几
```

**3.2 闰年判断与闰年统计**

```python
import calendar

# 判断某年是否是闰年
print(calendar.isleap(2026)) # False
print(calendar.isleap(2024)) # True

# 计算两个年份之间有多少个闰年（包含起始年，不包含结束年）
print(calendar.leapdays(2000, 2026))
```

**3.3 直接打印日历**
如果你需要在控制台打印出一个排版整齐的月历或年历，可以直接使用：

```python
import calendar
# 打印 2026 年 5 月的日历
print(calendar.month(2026, 5))
```

##### **4. datetime 与 time 模块通用的格式化占位符速查表**

在 Python 中，`datetime` 模块的 `strftime()` 方法与 `time` 模块的 `strftime()` 函数**共用同一套格式化占位符（格式码）**。这意味着下表不仅适用于 `datetime` 对象，同样完美适用于 `time` 模块中的 `struct_time` 元组。

**日期相关占位符**

| 占位符 | 含义           | 输出示例   | 备注                 |
| ------ | -------------- | ---------- | -------------------- |
| **%Y** | 四位数年份     | 2026       | 最常用               |
| **%y** | 两位数年份     | 26         | 仅显示年份后两位     |
| **%m** | 两位数月份     | 01 到 12   | 不足两位补零         |
| **%B** | 月份全名       | May        | 本地化名称（如英文） |
| **%b** | 月份缩写       | May        | 本地化缩写           |
| **%d** | 两位数日期     | 01 到 31   | 一个月中的第几天     |
| **%j** | 一年中的第几天 | 001 到 366 | 常用于年度统计       |

**时间相关占位符**

| 占位符 | 含义         | 输出示例         | 备注                      |
| ------ | ------------ | ---------------- | ------------------------- |
| **%H** | 24小时制小时 | 00 到 23         | 最常用                    |
| **%I** | 12小时制小时 | 01 到 12         | 需配合 %p 使用            |
| **%p** | AM 或 PM     | AM / PM          | 仅对12小时制有效          |
| **%M** | 两位数分钟   | 00 到 59         | 不足两位补零              |
| **%S** | 两位数秒钟   | 00 到 59         | 不足两位补零              |
| **%f** | 微秒         | 000000 到 999999 | 6位数字，常用于高精度记录 |

**星期、日历与时区相关占位符**

| 占位符 | 含义           | 输出示例 | 备注                                     |
| ------ | -------------- | -------- | ---------------------------------------- |
| **%A** | 星期全名       | Friday   | 本地化名称                               |
| **%a** | 星期缩写       | Fri      | 本地化缩写                               |
| **%w** | 星期几（数字） | 0 到 6   | **0代表星期日**（注意与`weekday()`区分） |
| **%u** | 星期几（数字） | 1 到 7   | **1代表星期一**（符合国内习惯）          |
| **%U** | 一年中的第几周 | 00 到 53 | 以**星期日**作为一周的第一天             |
| **%W** | 一年中的第几周 | 00 到 53 | 以**星期一**作为一周的第一天             |
| **%z** | UTC偏移量      | +0800    | 用于带时区的时间对象                     |
| **%Z** | 时区名称       | CST, UTC | 若未指定时区则为空字符串                 |
| **%%** | 百分号本身     | %        | 当字符串中需要显示 % 时使用              |

**补充验证：两者完全互通的代码示例**

```python
import time
from datetime import datetime

# 定义一套通用的格式化字符串
fmt = "%Y年%m月%d日 %H:%M:%S"

# 1. 在 datetime 模块中使用
now_dt = datetime.now()
print("datetime 格式化结果：", now_dt.strftime(fmt))

# 2. 在 time 模块中使用 (传入 struct_time)
now_time_struct = time.localtime()
print("time 模块格式化结果：", time.strftime(fmt, now_time_struct))
```

**使用小贴士：**

- **大小写敏感**：`%m` 是月份，`%M` 是分钟；`%H` 是24小时制，`%I` 是12小时制。
- **f-string 优雅写法**：在 Python 3.6+ 中，可以直接在 f-string 的大括号内使用占位符：`print(f"当前时间：{now:%Y年%m月%d日}")`。

##### **5. 避坑指南：时区与星期**

**星期几的“0”到底是谁？**

- `datetime` 模块中：`weekday()` 返回 0-6，**0 是星期一**。
- `calendar` 模块中：默认也是 0-6，**0 是星期一**（可以通过 `setfirstweekday` 修改）。
- `time` 模块的 `struct_time` 中：`tm_wday` 也是 0-6，**0 是星期一**。
- *注意：在 datetime 和 time 模块通用的格式化占位符 `%w` 中，0 代表星期日。在 Python 中只要记住“weekday() 和 struct_time 的 0 是周一”即可。*

**时区陷阱（Naive vs Aware）**
默认情况下，`datetime.now()` 生成的是一个“天真”（Naive）的时间对象，它不包含时区信息。如果你的项目涉及跨国业务（比如上海和纽约的服务器时间对比），强烈建议在 Python 3.9+ 中使用自带的 `zoneinfo`模块来生成带时区的时间对象，避免时间换算错误。

这三个模块配合起来，基本上能覆盖从底层计时、业务时间计算到日历展示的所有需求！如果在实际写代码时遇到具体的场景，随时可以拿来练手。

#### os模块

##### 1. 文件与目录管理

这是 `os` 模块最高频的使用场景，涵盖了创建、删除、重命名和遍历等操作。

- **获取与切换工作目录**

  - `os.getcwd()`：获取当前 Python 脚本执行所在的工作目录（绝对路径）。
  - `os.chdir(path)`：改变当前工作目录到指定的路径。

- **目录的增删查**

  - `os.listdir(path=".")`：列出指定目录下的所有文件和子目录名称（返回一个列表）。
  - `os.mkdir(path)`：创建单层目录。如果目录已存在，会抛出 `FileExistsError`。
  - `os.makedirs(path, exist_ok=True)`：递归创建多层目录（类似 Linux 的 `mkdir -p`）。加上 `exist_ok=True` 可以避免目录已存在时报错。
  - `os.rmdir(path)`：删除**空**目录。
  - `os.removedirs(path)`：递归删除空的目录链（从最深层开始向上删除）。

- **文件的操作**

  - `os.remove(path)`：删除指定路径的文件。
  - `os.rename(src, dst)`：重命名文件或目录，也可以用于移动文件（在同个磁盘下）。

- **递归遍历目录树**

  - `os.walk(top)`：这是遍历目录的利器。它会返回一个生成器，每次迭代产生一个三元组 `(root, dirs, files)`，分别代表当前遍历的目录路径、该目录下的子目录列表、该目录下的文件列表。

- **查看文件状态**

  - `os.stat(fpath)`：查看文件状态。它会返回一个 `os.stat_result` 对象，该对象包含了丰富的文件属性。

    | 属性名      | 含义说明                             | 备注                                                         |
    | ----------- | ------------------------------------ | ------------------------------------------------------------ |
    | st_mode     | 文件的类型和权限位掩码               | 需配合 `stat` 模块解析（如判断是否为目录/文件）              |
    | st_ino      | 索引节点号 (inode number)            | Windows 上表示文件索引号，Unix 上为 inode                    |
    | st_dev      | 文件所在设备的标识符                 | -                                                            |
    | st_nlink    | 硬链接的数量                         | -                                                            |
    | st_uid      | 文件所有者的用户 ID                  | -                                                            |
    | st_gid      | 文件所有者的组 ID                    | -                                                            |
    | st_size     | 普通文件的大小                       | 单位为字节 (bytes)                                           |
    | st_atime    | 最近一次访问时间 (Access Time)       | 以秒为单位的时间戳                                           |
    | st_mtime    | 最近一次内容修改时间 (Modified Time) | 最常用的时间戳，以秒为单位                                   |
    | st_ctime    | 平台相关的时间戳                     | Windows: 文件的创建时间 Unix: 最近一次元数据（权限等）更改时间 |
    | st_atime_ns | 最近一次访问时间                     | 以纳秒为单位的整数（现代 Python 版本支持）                   |
    | st_mtime_ns | 最近一次内容修改时间                 | 以纳秒为单位的整数                                           |
    | st_ctime_ns | 平台相关的时间戳                     | 以纳秒为单位的整数                                           |


**实战示例：遍历目录并找出所有 .py 文件**

```python
import os

py_files = []
# 遍历当前目录下的所有文件
for root, dirs, files in os.walk('.'):
    for file in files:
        if file.endswith('.py'):
            # 使用 os.path.join 拼接完整路径
            py_files.append(os.path.join(root, file))

print("找到的Python文件：", py_files)
```

##### 2. 路径处理（os.path 子模块）

`os.path` 是 `os` 模块的逻辑延伸，专门处理字符串级别的路径操作。它的最大价值在于**跨平台兼容性**，能自动适配 Windows（`\`）和 Linux/macOS（`/`）的路径分隔符差异。

- **路径拼接与拆分**
  - `os.path.join(path1, path2, ...)`：**强烈推荐使用**。智能拼接路径，自动处理不同系统的分隔符，避免手动拼接出错。
  - `os.path.split(path)`：将路径拆分为目录和文件名，返回元组 `(head, tail)`。
  - `os.path.splitext(path)`：将路径拆分为文件名和扩展名，返回元组 `(root, ext)`。
- **路径判断与获取信息**
  - `os.path.exists(path)`：检查路径（文件或目录）是否存在。
  - `os.path.isfile(path)` / `os.path.isdir(path)`：判断是否为文件或目录。
  - `os.path.abspath(path)`：返回路径的绝对路径。
  - `os.path.getsize(path)`：获取文件的大小（单位：字节）。
  - `os.path.getmtime(path)`：获取文件最后的修改时间（返回时间戳）。

**实战示例：跨平台安全拼接路径**

```python
import os

# 自动适配不同操作系统的路径分隔符
config_path = os.path.join("user", "data", "config.ini")
print(f"最终路径：{config_path}")  
# Windows 输出：user\data\config.ini
# Linux/Mac 输出：user/data/config.ini
```

##### 3. 环境变量与系统信息

`os` 模块可以方便地读取和设置操作系统的环境变量，并获取一些基础的系统和进程信息。

- **环境变量操作**
  - `os.environ`：一个包含所有环境变量的字典。
  - `os.getenv(key, default=None)`：安全地获取指定环境变量的值，如果不存在则返回默认值。
  - `os.environ['KEY'] = 'value'`：设置环境变量（仅在当前进程及其子进程中生效）。
- **系统与进程信息**
  - `os.name`：返回操作系统的名称，Windows 返回 `'nt'`，Linux/macOS 返回 `'posix'`。
  - `os.cpu_count()`：返回当前系统的 CPU 核心数。
  - `os.getpid()`：获取当前进程的 ID。
  - `os.system(command)`：在系统的 Shell 中执行命令（如 `ls -l` 或 `dir`），返回命令的退出状态码。

##### 4. 进阶避坑指南与最佳实践

+ **执行系统命令的替代方案**
  虽然 `os.system()` 简单易用，但它无法方便地获取命令的输出结果，且存在一定的安全隐患。在现代 Python 开发中，**强烈推荐使用 `subprocess` 模块**（如 `subprocess.run()`）来替代 `os.system()`，它能更安全、灵活地控制输入输出和错误处理。
+ **路径操作尽量使用 `os.path` 或 `pathlib`**
  永远不要手动使用字符串拼接路径（如 `path + '/' + file`），一定要使用 `os.path.join()`。此外，Python 3.4+ 引入了面向对象的 `pathlib` 模块，它提供了比 `os.path` 更优雅的路径操作方式，也是未来的推荐趋势。
+ **文件删除前的安全检查**
  在删除文件或目录前，最好先使用 `os.path.exists()` 判断路径是否存在，使用 `os.access(path, os.W_OK)`检查是否有写入/删除权限，避免程序因权限不足或路径不存在而崩溃。

#### shutil 模块

`shutil`（Shell Utilities 的缩写）是 Python 标准库中专门用于**高级文件操作**的工具箱。如果说 `os` 模块主要负责底层的路径处理和简单的系统交互，那么 `shutil` 就是帮你解决那些在命令行里经常用到、但自己写起来又特别麻烦的复杂文件任务（比如复制整个文件夹、跨磁盘移动文件、打包压缩等）。

以下是 `shutil` 模块最核心、最高频使用的功能总结：

**1. 强大的文件与目录复制**

`shutil` 提供了多种层级的复制功能，你可以根据是否需要保留文件的“元数据”（如修改时间、权限等）来灵活选择：

- **`shutil.copyfile(src, dst)`**：**只复制内容**。它是最基础的复制，仅将源文件的字节流复制到目标文件，不保留任何权限、时间戳等元数据。
- **`shutil.copy(src, dst)`**：**复制内容 + 权限**。除了文件内容外，还会把原文件的权限模式也复制过去。
- **`shutil.copy2(src, dst)`**：**完美克隆（强烈推荐）**。在 `copy` 的基础上，进一步保留了文件的完整元数据（如修改时间、访问时间等）。在做文件备份时，这通常是首选方法。
- **`shutil.copytree(src, dst)`**：**递归复制整个目录树**。它可以一键将整个文件夹及其内部的所有子文件夹、文件全部复制过去。从 Python 3.8 开始，增加了 `dirs_exist_ok=True` 参数，允许目标文件夹已存在时继续执行复制。

```python
import shutil

# 1. 完美克隆单个文件（保留修改时间等信息）
shutil.copy2('source.txt', 'backup.txt')

# 2. 递归复制整个文件夹（如果目标已存在则合并）
shutil.copytree('my_project', 'my_project_backup', dirs_exist_ok=True)
```

**2. 智能的文件移动与删除**

- **`shutil.move(src, dst)`**：**智能移动文件/目录**。它的行为非常聪明：如果在同一个磁盘分区内移动，它会直接重命名（毫秒级完成）；如果是跨磁盘或跨分区移动，它会自动执行“先复制过去，再删除原文件”的操作，完全不需要你手动判断。
- **`shutil.rmtree(path)`**：**暴力递归删除**。普通的 `os.rmdir()` 只能删空文件夹，而 `rmtree` 可以连同里面的所有子文件和子文件夹一起彻底删除（相当于命令行的 `rm -rf`）。**⚠️注意：此操作不可逆，没有回收站，使用时务必确认路径正确！**

```python
import shutil

# 移动文件（支持跨磁盘）
shutil.move('old_folder/data.zip', 'new_folder/')

# 彻底删除一个非空目录
shutil.rmtree('temp_cache')
```

**3. 一键打包压缩与解压**

`shutil` 封装了底层的 `zipfile` 和 `tarfile` 模块，让你可以用极少的代码实现文件夹的压缩和解压：

- **`shutil.make_archive(base_name, format, root_dir)`**：创建压缩包。支持 `zip`, `tar`, `gztar` (tar.gz), `bztar`(tar.bz2) 等多种格式。
- **`shutil.unpack_archive(filename, extract_dir)`**：自动识别格式并解压文件。

```python
import shutil

# 将 my_data 文件夹打包成 my_data.zip
shutil.make_archive('my_data', 'zip', 'my_data')

# 将压缩包解压到指定的 output 文件夹
shutil.unpack_archive('my_data.zip', 'output')
```

**4. 进阶实用技巧**

1. **大文件安全复制 (`copyfileobj`)**：如果你需要处理几个 G 的超大文件，直接使用 `copy2` 可能会占用较多内存。可以使用 `shutil.copyfileobj(fsrc, fdst)`，它接受已经打开的文件对象，以分块缓冲的方式读写，非常节省内存。
2. **查询磁盘空间 (`disk_usage`)**：通过 `shutil.disk_usage('/')` 可以快速获取指定路径所在磁盘的总容量、已用空间和剩余空间（返回字节数），在做自动化脚本时非常有用。
3. **配合 `os` 模块使用**：在实际开发中，通常会将 `shutil` 和 `os` 模块搭配使用。例如，先用 `os.path.exists()`检查路径是否存在，再用 `shutil` 进行复制或移动，这样能让你的文件处理脚本更加健壮和安全。

#### collections 模块

Python 的 `collections` 模块是标准库中提供**高性能容器数据类型**的宝藏工具。它提供了许多比内置容器（如 `list`、`dict`、`set`）更强大、更专业的替代选择，能够帮助我们写出更高效、更优雅的 Python 代码。

以下是 `collections` 模块中最核心、最高频使用的 5 大组件详解：

##### 1. defaultdict：告别“键不存在”的报错

`defaultdict` 是一个自带默认值的字典。当访问一个不存在的键时，它会自动调用传入的“工厂函数”生成一个默认值，而不是抛出 `KeyError`。这在处理计数或分组逻辑时极其好用。

- **核心优势**：省去了每次操作前都要写 `if key not in dict` 的繁琐判断。
- **常用工厂函数**：`int`（默认值为0）、`list`（默认值为空列表）、`set`（默认值为空集合）。

**实战示例：统计单词出现次数**

```python
from collections import defaultdict

words = ["apple", "banana", "apple", "orange", "banana", "apple"]

# ❌ 传统字典写法：每次都要判断
count = {}
for word in words:
    if word not in count:
        count[word] = 0
    count[word] += 1

# ✅ defaultdict 写法：自动初始化默认值 0
count = defaultdict(int)  
for word in words:
    count[word] += 1  # 即使键不存在，也会自动初始化为 0 再执行 +=1

print(dict(count))  # 输出: {'apple': 3, 'banana': 2, 'orange': 1}
```

##### 2. Counter：统计学的瑞士军刀

`Counter` 是一个专门用于计数的字典子类。它可以轻松统计可哈希对象（如字符串、列表中的元素）的出现次数，并提供了许多便捷的统计方法。

- **核心方法**：
  - `most_common(n)`：返回出现次数最多的前 n 个元素及其计数。
  - 支持数学运算：可以直接对两个 `Counter` 对象进行加减操作（如 `c1 + c2`）。

**实战示例：快速找出出现频率最高的元素**

```python
from collections import Counter

words = ['apple', 'banana', 'apple', 'orange', 'banana', 'apple']
word_counts = Counter(words)

# 统计结果
print(word_counts)  # 输出: Counter({'apple': 3, 'banana': 2, 'orange': 1})

# 获取出现次数最多的前 2 个
top2 = word_counts.most_common(2)
print(top2)  # 输出: [('apple', 3), ('banana', 2)]

# 检测两个字符串是否为字母异位词（字符组成完全相同）
print(Counter('listen') == Counter('silent'))  # 输出: True
```

**3. deque：高性能双端队列**

`deque`（double-ended queue）是一个线程安全的双端队列。它的底层采用双向链表结构，**在队列的两端进行插入和删除操作的时间复杂度都是 O(1)**，完美解决了普通 `list` 在头部操作（如 `pop(0)`）时效率极低（O(n)）的性能瓶颈。

- **核心特性**：
  - 支持 `append()` / `pop()`（右端操作）和 `appendleft()` / `popleft()`（左端操作）。
  - 可以设置 `maxlen` 参数，当队列满时，新元素的加入会自动淘汰旧元素，非常适合实现滑动窗口或固定长度的历史记录。

**实战示例：维护固定长度的历史记录**

```python
from collections import deque

# 创建一个最大长度为 3 的双端队列
history = deque(maxlen=3)

for i in range(5):
    history.append(f"Log {i}")

# 超出长度后，最早的数据会自动被挤出
print(list(history))  # 输出: ['Log 2', 'Log 3', 'Log 4']
```

**4. namedtuple：类型安全的元组替代**

`namedtuple` 用于创建一个带有命名字段的元组子类。它既保留了普通元组**不可变**和**高性能**的特性，又可以通过属性名（而不是下标索引）来访问元素，极大地提高了代码的可读性。

- **适用场景**：替代那些只有属性没有方法的简单类，或者处理数据库查询返回的记录。

**实战示例：用名字代替索引访问元组**

```python
from collections import namedtuple

# 定义一个名为 Point 的命名元组，包含 x 和 y 两个字段
Point = namedtuple('Point', ['x', 'y'])
p = Point(11, y=22)

# 既支持通过属性名访问（推荐），也支持通过索引访问
print(p.x)       # 输出: 11
print(p[1])      # 输出: 22

# 转换为字典
print(p._asdict())  # 输出: {'x': 11, 'y': 22}
```

**5. OrderedDict：明确维护插入顺序的字典**

`OrderedDict` 是一个能够记住元素插入顺序的字典。虽然 Python 3.7+ 的普通 `dict` 已经保证了插入顺序，但 `OrderedDict` 依然有其独特的价值。

- **独特价值**：它提供了普通字典没有的**顺序控制方法**，例如 `move_to_end(key)`（将某个键移动到末尾）和 `popitem(last=False)`（弹出最早插入的项）。这些特性使其成为实现 **LRU（最近最少使用）缓存**的绝佳基石。

**实战示例：调整字典元素的顺序**

```python
from collections import OrderedDict

od = OrderedDict()
od['a'] = 1
od['b'] = 2
od['c'] = 3

# 将键 'a' 移动到字典的末尾
od.move_to_end('a')
print(od)  # 输出: OrderedDict([('b', 2), ('c', 3), ('a', 1)])
```

#### random 模块：随机数生成器

`random` 模块主要用于生成各种类型的伪随机数，广泛应用于游戏开发、数据采样、A/B 测试等场景。

- **生成随机数**：
  - `random.random()`：生成一个 [0, 1) 之间的随机浮点数。
  - `random.randint(a, b)`：生成一个 [a, b] 闭区间内的随机整数。
  - `random.uniform(a, b)`：生成一个 [a, b] 区间内的随机浮点数。
- **序列操作**：
  - `random.choice(seq)`：从非空序列中随机抽取一个元素。
  - `random.sample(population, k)`：从序列中无重复地随机抽取 k 个元素。
  - `random.shuffle(seq)`：将序列原地打乱顺序（洗牌）。
- **可重复性（种子）**：
  - `random.seed(value)`：设置随机数种子。在测试或需要结果可复现的科学实验中，固定种子可以保证每次运行生成的随机序列完全一致。

#### math 模块：高精度数学运算

`math` 模块提供了对 C 标准定义的通用数学函数和常量的访问，适用于需要高精度浮点运算的场景。

- **数学常数**：
  - `math.pi`：圆周率 π（约 3.141592）。
  - `math.e`：自然对数的底数 e（约 2.718281）。
- **常用数学函数**：
  - **数值处理**：`math.ceil(x)`（向上取整）、`math.floor(x)`（向下取整）、`math.fabs(x)`（取绝对值）。
  - **幂与对数**：`math.sqrt(x)`（平方根）、`math.pow(x, y)`（x的y次幂）、`math.log(x)`（自然对数）。
  - **三角函数**：`math.sin(x)`、`math.cos(x)` 等（注意：传入的参数必须是弧度，可使用 `math.radians()`将角度转为弧度）。
  - **数论相关**：`math.factorial(x)`（阶乘）、`math.gcd(a, b)`（最大公约数）。

#### statistics 模块：基础数据统计

`statistics` 模块专注于提供计算数字（Real-valued）数据的数学统计属性的函数，非常适合进行基础的数据分析。

- **集中趋势度量**：
  - `statistics.mean(data)`：计算算术平均数。
  - `statistics.median(data)`：计算中位数。
  - `statistics.mode(data)`：计算众数（出现次数最多的数据）。
- **离散程度度量**：
  - `statistics.stdev(data)`：计算样本标准差。
  - `statistics.variance(data)`：计算样本方差。
- **相关性**：
  - `statistics.correlation(x, y)`：计算两组数据的皮尔逊相关系数。

#### sys 模块：与解释器深度交互

`sys` 模块提供了访问和维护 Python 解释器运行时环境的各种变量和函数，是编写系统级或需要深度控制程序行为的必备工具。

- **命令行参数**：`sys.argv` 是一个列表，存储了从命令行传递给 Python 脚本的所有参数（`argv[0]` 是脚本名，后续为传入的参数）。
- **标准输入输出**：`sys.stdin`、`sys.stdout`、`sys.stderr` 分别对应标准输入、输出和错误流对象。
- **程序控制**：`sys.exit(arg)` 用于退出当前程序。传入 0 表示正常退出，非 0 或字符串表示异常退出。
- **路径与模块**：`sys.path` 是一个列表，包含了 Python 查找模块的路径集合（可以动态添加自定义路径）。
- **系统信息**：`sys.platform` 返回操作系统标识（如 win32, linux, darwin）；`sys.version` 返回 Python 解释器的版本信息。

#### string 模块：字符串常量与处理工具

虽然 Python 的字符串对象（`str`）自带了丰富的方法，但 `string` 模块提供了一些预定义的字符串常量和传统的模板功能。

- **常用字符串常量**：在需要生成随机密码、验证码或进行字符过滤时，这些常量非常高效。
  - `string.ascii_letters`：所有大小写英文字母（abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ）。
  - `string.digits`：数字 0-9。
  - `string.punctuation`：所有特殊符号字符（如 !"#$%&'()*+,-./ 等）。
- **字符串模板**：`string.Template` 提供了一种简化的字符串替换语法，适合让用户在不了解复杂 Python 语法的情况下编辑模板文本。

#### 序列化与反序列化模块

在 Python 的世界里，数据通常以变量的形式存在于内存中。一旦程序结束，这些数据就会消失。**序列化（Serialization）**就是解决这个问题的关键技术——它能将内存中的对象转换为可以存储或传输的格式（如字节流或字符串），而**反序列化（Deserialization）**则是将其还原回来的过程。

Python 中最常用的两个模块是 `json` 和 `pickle`。虽然它们的目标相似，但适用场景截然不同。本文将带你深入理解这两个模块的异同与用法。

**核心对比：JSON vs Pickle**

在深入代码之前，我们先通过一个表格直观地对比这两个模块的核心特性：

| 特性         | `json` 模块                      | `pickle` 模块                         |
| ------------ | -------------------------------- | ------------------------------------- |
| **数据格式** | 文本 (人类可读)                  | 二进制 (字节流)                       |
| **跨语言**   | ✅ 通用 (JavaScript, Java等)      | ❌ 仅限 Python                         |
| **安全性**   | 高 (相对安全)                    | 低 (不要反序列化不可信数据)           |
| **支持对象** | 基础数据类型 (dict, list, str等) | 几乎所有 Python 对象 (函数、类实例等) |
| **主要用途** | API 接口、配置文件、Web 数据交换 | Python 对象持久化、缓存               |

##### JSON 模块：通用的数据交换语言

JSON（JavaScript Object Notation）是一种轻量级的数据交换格式。`json` 模块主要用于处理结构化数据，是 Web 开发和 API 交互的事实标准。

**核心函数**

- **`json.dumps(obj)`**: 将 Python 对象**序列化**为 JSON 字符串。
- **`json.loads(str)`**: 将 JSON 字符串**反序列化**为 Python 对象。
- **`json.dump(obj, file)`**: 将对象序列化并写入文件对象。
- **`json.load(file)`**: 从文件对象中读取 JSON 数据并反序列化。

**代码示例：数据存储与读取**

```python
import json

# 1. 定义一个复杂的数据结构
data = {
    "name": "Alice",
    "age": 30,
    "hobbies": ["reading", "coding"],
    "is_student": False,
    "address": None
}

# 2. 序列化：写入文件
with open("data.json", "w", encoding="utf-8") as f:
    # ensure_ascii=False 保证中文不被转义
    # indent=4 让文件格式美观（人类可读）
    json.dump(data, f, ensure_ascii=False, indent=4)

# 3. 反序列化：读取文件
with open("data.json", "r", encoding="utf-8") as f:
    loaded_data = json.load(f)
    print(loaded_data["name"])  # 输出: Alice
```

**注意事项**

1. **类型限制**：JSON 只支持基础类型。如果你尝试序列化 `datetime` 对象或自定义类，会报错 `Object of type ... is not JSON serializable`。
2. **编码**：处理中文时，务必在 `open` 时指定 `encoding="utf-8"`，并在 `dump` 时设置 `ensure_ascii=False`，否则中文会变成 Unicode 转义字符。

##### Pickle 模块：Python 的专属记忆胶囊

Pickle 是 Python 的“原生”序列化工具。它不仅能处理基础数据类型，还能完美保存 Python 的**函数、类定义、实例状态**等复杂对象。

**核心函数**

- **`pickle.dumps(obj)`**: 序列化为字节对象。
- **`pickle.loads(bytes)`**: 从字节对象反序列化。
- **`pickle.dump(obj, file)`**: 序列化到文件。
- **`pickle.load(file)`**: 从文件反序列化。

**代码示例：保存函数与自定义对象**

```python
import pickle

# 1. Pickle 可以序列化函数！
def greet(name):
    return f"Hello, {name}!"

# 2. 定义一个自定义类
class Person:
    def __init__(self, name):
        self.name = name
    
    def say_hello(self):
        return f"Hi, I'm {self.name}"

# 3. 序列化复杂对象
p = Person("Bob")

with open("data.pkl", "wb") as f: # 注意：必须用二进制写模式 'wb'
    pickle.dump(greet, f)   # 保存函数
    pickle.dump(p, f)       # 保存实例

# 4. 反序列化
with open("data.pkl", "rb") as f: # 注意：必须用二进制读模式 'rb'
    loaded_func = pickle.load(f)
    loaded_person = pickle.load(f)
    
    print(loaded_func("World"))        # 输出: Hello, World!
    print(loaded_person.say_hello())   # 输出: Hi, I'm Bob
```

**安全警告**

**Pickle 是不安全的！** Pickle 反序列化过程实际上是在执行 Python 代码。如果加载了来自不可信来源的 Pickle 文件，可能会导致任意代码执行（RCE）。**永远不要反序列化你不信任的数据。**

**实战场景建议**

1. **用 JSON 的场景**：
   - 你需要将数据发送给前端网页（JavaScript）。
   - 你需要编写配置文件（如 `.json` 配置）。
   - 你需要数据在 Python、Java、Go 等多种语言之间共享。
   - 你需要人类能直接阅读和修改文件内容。
2. **用 Pickle 的场景**：
   - 你需要保存机器学习模型（如 Scikit-learn 模型）。
   - 你需要缓存复杂的 Python 计算结果（如大型 DataFrame）。
   - 你需要保存 Python 程序的运行状态（如游戏存档）。
   - 数据不需要被其他语言读取。

#### configparser模块

**1. 配置文件的基本格式**

`.ini` 文件由**节（Section）**和**键值对（Option/Value）**组成。

- **节（Section）**：用方括号 `[]` 包围，用于对配置项进行分组。
- **键值对（Option/Value）**：在节下面，用 `=` 或 `:` 分隔键和值。
- **注释**：使用 `#` 或 `;` 开头，必须独立成行。

**示例配置文件 (`config.ini`)：**

```ini
[database]
; 数据库配置
host = localhost
port = 3306
user = admin
password = secret123

[logging]
level = INFO
file = app.log
```

**2. 核心操作与常用方法**

使用 `configparser` 的核心是创建一个 `ConfigParser` 对象。

**基本读取与写入：**

```python
import configparser

# 1. 创建解析器对象
config = configparser.ConfigParser()

# 2. 读取配置文件（支持指定编码）
config.read('config.ini', encoding='utf-8')

# 3. 修改或新增配置项
config.set('database', 'host', '127.0.0.1')  # 修改
config.set('database', 'timeout', '30')      # 新增

# 4. 将修改写回文件
with open('config.ini', 'w', encoding='utf-8') as f:
    config.write(f)
```

**常用方法速查表：**

| 方法                      | 作用                                 | 示例                                          |
| ------------------------- | ------------------------------------ | --------------------------------------------- |
| `sections()`              | 获取所有节（Section）的名称列表      | `config.sections()`                           |
| `options(section)`        | 获取指定节下的所有键名               | `config.options('database')`                  |
| `items(section)`          | 获取指定节下的所有键值对（元组列表） | `config.items('database')`                    |
| `has_section(sec)`        | 检查是否存在某个节                   | `config.has_section('database')`              |
| `has_option(sec, opt)`    | 检查指定节下是否存在某个键           | `config.has_option('database', 'host')`       |
| `add_section(sec)`        | 添加一个新的节                       | `config.add_section('new_section')`           |
| `remove_section(sec)`     | 删除指定的节                         | `config.remove_section('new_section')`        |
| `remove_option(sec, opt)` | 删除指定节下的某个键                 | `config.remove_option('database', 'timeout')` |

**3. 获取数据与类型转换**

在 `.ini` 文件中，所有的值默认都是以**字符串**形式存储的。`configparser` 提供了便捷的方法来获取并自动转换数据类型：

- **获取字符串**：`config.get('database', 'host')`
- **获取整数**：`config.getint('database', 'port')`
- **获取浮点数**：`config.getfloat('section', 'rate')`
- **获取布尔值**：`config.getboolean('section', 'debug')`（支持 `yes/no`, `true/false`, `1/0` 等）

**安全获取与默认值：**
如果配置的键可能不存在，可以使用 `fallback` 参数设置默认值，避免程序报错：

```python
# 如果 'timeout' 不存在，则返回默认值 10
timeout = config.getint('database', 'timeout', fallback=10)
```

**4. 重要的注意事项与最佳实践**

- **大小写敏感问题**：
  - **节（Section）**的名称是**大小写敏感**的。
  - **键（Option）**的名称默认是**大小写不敏感**的。`configparser` 在内部会自动将键名转换为小写。因此，`config['database']['HOST']` 和 `config['database']['host']` 指向的是同一个值。
- **特殊的 `[DEFAULT]` 节**：
  - 配置文件中可以有一个名为 `[DEFAULT]` 的特殊节。它里面的键值对会作为**全局默认值**，提供给其他所有的节使用。
- **注释丢失问题**：
  - 使用 `config.write()` 将配置写回文件时，**原始文件中的注释会被全部清除**。因此，如果配置文件中有重要的人工注释，直接程序化写入可能会导致注释丢失。
- **错误处理**：
  - 在读取不存在的节或键时，会分别抛出 `NoSectionError` 和 `NoOptionError` 异常，建议在关键配置读取时做好异常捕获或使用 `fallback` 默认值。

### 进阶概念：包（Package）

当自定义模块越来越多时，我们需要用文件夹来进一步管理它们。在 Python 中，包含 `__init__.py` 文件的文件夹就被称为“包”。 包可以包含多个模块，甚至包含子包，从而构建出层次分明的项目结构。例如：

```text
my_project/
├── main.py
└── utils/           # 这是一个包
    ├── __init__.py  # 包的标识文件
    └── file_utils.py # 包内的一个模块
```

#### 避坑指南：`__name__` 与循环导入

在学习模块时，有两个经典问题需要特别注意：

- **`if __name__ == '__main__':` 的作用** 每个模块都有一个内置属性 `__name__`。当该文件被直接运行时，`__name__` 的值为 `'__main__'`；当它被作为模块导入时，`__name__` 的值为该模块的文件名。我们通常利用这个特性来编写测试代码，确保这些代码只在直接运行该文件时执行，而在被导入时不执行。

  `if __name__ == '__main__':` 是 Python 中一个非常经典且极其重要的代码模式。它的核心作用相当于一个**“程序入口的安检门”**，用来判断当前这个 Python 文件是被**直接运行**的，还是被其他文件**作为模块导入**的。

  **💡 为什么要用它？（核心作用）**

  在 Python 中，任何一个 `.py` 文件都有两种使用方式：

  1. **作为脚本直接运行**（比如在命令行输入 `python my_script.py`）。
  2. **作为模块被其他文件导入**（比如在其他文件中写 `import my_script`）。

  `if __name__ == '__main__':` 的作用就是：**让缩进块内的代码，只在第1种情况（直接运行）下执行，而在第2种情况（被导入）时自动跳过**。

  这样做最大的好处是实现了**“一鱼两吃”**：既能把文件当作独立的程序来跑，又能把它当作一个安全的工具库（模块）给别人调用，而不会因为导入就意外触发一堆测试代码或主逻辑。

  **⚙️ 底层原理是什么？**

  Python 解释器在执行任何文件时，都会自动创建一个名为 `__name__` 的特殊内置变量。这个变量的值取决于文件的运行方式：

  - **直接运行时**：解释器会把 `__name__` 的值设置为字符串 `"__main__"`。
  - **被导入时**：解释器会把 `__name__` 的值设置为该模块的实际文件名（不带 `.py` 后缀）。

  因此，`if __name__ == '__main__':` 这句话翻译成大白话就是：“**如果当前文件是作为主程序启动的，那就执行下面的代码；否则（如果是被别人借用的），就什么都别做。**”

  **💻 直观的代码演示**

  假设我们有一个名为 `demo.py` 的文件，内容如下：

  ```python
  # demo.py
  def say_hello(name):
      print(f"你好，{name}！")
  
  # 只有直接运行 demo.py 时，才会打印下面这行字并调用函数
  if __name__ == '__main__':
      print("我是直接被运行的！")
      say_hello("Jabin")
  ```

  **场景一：直接在终端运行它**
  执行命令 `python demo.py`，输出结果为：

  ```text
  我是直接被运行的！
  你好，Jabin！
  ```

  *(此时 `__name__` 的值为 `"__main__"`，条件成立，代码正常执行)*

  **场景二：在另一个文件 `main.py` 中导入它**
  我们在同级目录下新建一个 `main.py`，写入：

  ```python
  # main.py
  import demo
  
  print("--- 我是 main.py ---")
  demo.say_hello("开发者")
  ```

  执行命令 `python main.py`，输出结果为：

  ```text
  --- 我是 main.py ---
  你好，开发者！
  ```

  *(注意！`demo.py` 里的 `print("我是直接被运行的！")` 并没有被执行。因为导入时 `demo.py` 的 `__name__` 变成了 `"demo"`，不等于 `"__main__"`，所以那个代码块被完美跳过了)*

  **🛠️ 最佳实践写法**

  在实际的项目开发中，为了保持代码结构清晰，我们通常不会把所有逻辑直接堆在 `if` 语句下面，而是先定义一个 `main()` 函数，然后在入口处调用它。这是最标准的工程化写法：

  ```python
  import sys
  
  def main():
      # 在这里编写你的核心业务逻辑
      print("程序正式启动...")
      # 处理参数、执行业务等
  
  if __name__ == '__main__':
      # 仅在直接运行时调用主函数
      main()
  ```

- **循环导入问题** 如果模块 A 导入了模块 B，而模块 B 又反过来导入了模块 A，就会形成“死循环”，导致程序报错。 **解决方案**：

  1. 优化代码结构，将互相依赖的公共部分提取到第三个模块中。
  2. 将导入语句放在函数内部（局部导入），延迟导入时机来打破循环。

掌握模块与包的使用，是迈向 Python 工程化开发的重要一步。合理地拆分与组织代码，不仅能让你的程序更加健壮，也能极大提升后续的开发与维护效率。
